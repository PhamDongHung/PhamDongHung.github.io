Các câu lệnh trong Java: Chi tiết từ cơ bản đến nâng cao

23/10/2025 5-minute read
Phạm Đông Hưng
Cú Pháp • Trình Diễn Chủ Đề
Chuyên Môn IT • Tài Liệu Java • Java

Việc làm chủ các câu lệnh (statements) là bước đi nền tảng và không thể thiếu đối với bất kỳ lập trình viên Java nào. Các câu lệnh này là các khối xây dựng cơ bản, quyết định cách chương trình hoạt động, từ việc quản lý dữ liệu đến điều khiển luồng thực thi. Bài viết này sẽ đi sâu vào các nhóm câu lệnh chính: cơ bản, điều khiển luồng, xử lý ngoại lệ, và đặc biệt, giúp bạn tối ưu hóa logic chương trình.

I. Phân Loại Câu Lệnh Java Cơ Bản Link to heading

Các câu lệnh cơ bản trong Java tập trung vào việc định nghĩa, lưu trữ và tương tác với dữ liệu.

1. Câu Lệnh Khai Báo Biến (Declaration Statement) Link to heading

Câu lệnh này dùng để định nghĩa một vùng nhớ (biến) trong bộ nhớ máy tính để lưu trữ dữ liệu.

  • Cú pháp: <Kiểu_Dữ_Liệu> <Tên_Biến>;
  • Ví dụ: 
    1
2
3

    int soNguyen;
double giaTriThuc;
String tenNguoiDung;
  • Giải thích: int, double, String xác định loại dữ liệu mà biến đó sẽ lưu trữ.

2. Câu Lệnh Gán Giá Trị (Assignment Statement) Link to heading

Dùng để cấp một giá trị cụ thể cho biến đã được khai báo.

  • Cú pháp: <Tên_Biến> = <Giá_Trị>;
  • Ví dụ: 
    1
2
3

    soNguyen = 42;
giaTriThuc = 19.99;
tenNguoiDung = "Le Van B";
  • Lưu ý: Giá trị gán phải tương thích với kiểu dữ liệu của biến.

3. Câu Lệnh In ra Màn hình (Output Statement) Link to heading

Sử dụng để hiển thị thông tin, kết quả tính toán hoặc thông báo ra console.

  • Cú pháp phổ biến: System.out.println(<Nội_Dung>); (In và tự động xuống dòng)
  • Ví dụ: 
    1
2

    System.out.println("Xin chao!");
System.out.print("Ket qua: " + (5 + 3));

4. Câu Lệnh Nhập Liệu từ Bàn Phím (Input Statement) Link to heading

Cho phép chương trình tương tác và thu thập dữ liệu do người dùng nhập vào thông qua lớp Scanner.

  • Các bước chính:
    • Import: import java.util.Scanner;
    • Khởi tạo đối tượng: Scanner sc = new Scanner(System.in);
    • Đọc dữ liệu: int tuoi = sc.nextInt();
  • Quan trọng: Nên gọi sc.close() khi kết thúc để tránh rò rỉ tài nguyên.

II. Các Câu Lệnh Điều Khiển Luồng (Flow Control) Link to heading

Nhóm câu lệnh này xác định thứ tự và điều kiện thực thi các đoạn mã, là cốt lõi của logic chương trình.

1. Câu Lệnh Điều Kiện (Conditional Statements) Link to heading

Loại Lệnh Mục Đích Cú Pháp Ví dụ
if Thực thi khối mã nếu điều kiện Đúng (true). if (điều_kiện) { ... } if (diem >= 50)
if-else Thực thi khối mã nếu điều kiện Đúng hoặc khối mã khác nếu Sai (false). if (cđ) { ... } else { ... } if (gio < 12) { chaoSang(); } else { chaoToi(); }
switch-case Kiểm tra một biến với nhiều trường hợp cố định. switch (biến) { case g1: ... break; default: ... } Thường dùng để kiểm tra các giá trị như int, String, enum.

2. Câu Lệnh Vòng Lặp (Looping Statements) Link to heading

Loại Vòng Lặp Khi nào dùng? Cú Pháp Đặc Điểm
for Khi biết trước số lần lặp. for (khởi_tạo; điều_kiện; bước_nhảy) { ... } Phù hợp cho việc duyệt mảng, lặp cố định.
while Khi chưa biết trước số lần lặp, cần điều kiện dừng. while (điều_kiện) { ... } Lặp lại 0 lần trở lên.
do-while Khi muốn khối mã được thực thi ít nhất một lần. do { ... } while (điều_kiện); Điều kiện được kiểm tra sau lần lặp đầu tiên.

3. Câu Lệnh Rẽ Nhánh Trong Vòng Lặp Link to heading

  • break: Dừng và thoát ngay lập tức khỏi vòng lặp hoặc khối switch gần nhất.
  • continue: Bỏ qua phần còn lại của lần lặp hiện tại và chuyển sang lần lặp tiếp theo.

III. Câu Lệnh Xử Lý Ngoại Lệ (Exception Handling) 🛡️ Link to heading

Giúp chương trình chạy ổn định bằng cách kiểm soát các lỗi (ngoại lệ) phát sinh trong quá trình thực thi.

Khối Lệnh/Từ Khóa Mục Đích Cú Pháp
try-catch Bắt và xử lý các ngoại lệ để chương trình không bị dừng đột ngột. try { ... } catch (LoaiNgoaiLe e) { ... }
finally Chứa khối mã luôn luôn thực thi sau khi try và catch kết thúc (thường dùng để đóng tài nguyên). try { ... } finally { ... }
throw Được dùng để ném một đối tượng ngoại lệ cụ thể theo logic lập trình. throw new LoaiNgoaiLe("Thông báo lỗi");
  • Ví dụ (try-catch-finally): 
    1
2
3
4
5
6
7

    try {
    int ketQua = 10 / 0; // Gây ra lỗi ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Loi: Khong the chia cho 0.");
} finally {
    System.out.println("Khoi nay luon chay.");
}

IV. Các Câu Lệnh Đặc Biệt Link to heading

Java còn có các câu lệnh đặc thù, phục vụ các mục đích nâng cao hơn:

1. Câu Lệnh return (Trả Giá Trị Về) Link to heading

Dùng để kết thúc việc thực thi một phương thức và, nếu phương thức đó có kiểu trả về, nó sẽ trả lại giá trị cho nơi gọi.

  • Ví dụ: 
    1
2
3
4
5
6
7

    public int tinhTong(int x, int y) {
  return x + y; // Trả về tổng
}
public void hienThi() {
  //... code ...
  return; // Kết thúc phương thức (tùy chọn trong void method)
}

2. Câu Lệnh assert (Kiểm Tra Giả Định) Link to heading

Dùng để kiểm tra một giả định phải đúng tại một thời điểm nào đó trong code. Nếu điều kiện sai, nó ném ra AssertionError. Chủ yếu dùng cho quá trình phát triển và debug.

  • Cú pháp: assert biểu_thức_logic : "Thông báo lỗi";

3. Câu Lệnh synchronized (Đồng Bộ Hóa) Link to heading

Cực kỳ quan trọng trong lập trình đa luồng (multithreading). Nó đảm bảo rằng chỉ một luồng được phép truy cập vào đoạn mã hoặc phương thức được đánh dấu tại một thời điểm, tránh xung đột dữ liệu.

  • Cú pháp: public synchronized void tangGiaTri() { ... } hoặc synchronized (doi_tuong_lock) { ... }

Việc nắm vững các loại câu lệnh này là chìa khóa để xây dựng các ứng dụng Java hiệu quả, ổn định và dễ bảo trì.
